1.7 - Describe which are the appropriate filetypes for websites

There are many more file types suitable for use on a website than those I've listed below, actually, so much so that if I were to list all the file types that could technically be used it would be an exhaustive list so here are seven for you read up on:

PNG - This image file type is one of my preferred types to use due to the higher overall quality (at the cost of file size) and extra options you simply don't get with image files types such as .JPG. The chief option that comes to mind is transparency like with my Twitch emotes which makes them useful for adjusting to the users settings of dark or light mode on their browser.

JPG - This is one of the most common file types used for images because lower general file size but this comes at the cost of detail when compared to .PNG.

GIF - Leaving the pronunciation of it aside which I assure you is a debate almost as old as time itself... .GIF is an image container that's designed to display images in an animated fashion to produce moving images. This format doesn't support sound however the newer format below does!

WEBM - This is a format similar to GIF in that it allows for animated images but unlike GIF it also allows for the playback of sound. Generally the quality of the images are better too albeit at the cost of file size as one may expect with this kind of multimedia.

GIT - This is a file that helps to keep our folder structures in-sync in the cloud using github so we can make and push the changes needed to effectively work on digital projects.

HTML - The HTML file type is used to save all of the HTML code used to create the framework of the website. This is a markup language and as such is vital or else you wouldn't be able to view this very website you're viewing right now!

CSS - CSS is used in tandem with HTML to style the websites. This makes it a style language and not a markup language unlike HTML which is used mostly for the content.
