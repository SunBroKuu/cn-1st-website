Describing the purpose of IT in modern business

I firmly believe the role of IT in the modern business place is to streamline things and open up new opportunities that otherwise wouldn’t be present to the companies that invest in the sector. There are certain sectors such as security that have been drastically enhanced by intelligent applications and hardware that make securing the things they need to much easier.

The bottom line is that time is money and the more a company invests in efficient design of their IT system the more time and money will be saved later down the line.

Describe the risks that might impact digital projects – what causes digital projects to fail? Why?

There are several risks that we could go into but let's focus on a security point of view and a time/money focus.

Security: Having an all digital protect could be dangerous due to malicious 3rd parties that are attempting to breach the defences to steal source code or other copyrighted material for their own ends.

Time & Money: The biggest cause for projects to fail is due to the lack of proper oversight and planning for unforeseen eventualities like extend time to implement parts of the project. Luckily with tools like Trello this shouldn’t happen all that often with proper management! Describe how you would go about selecting and using IT systems and software How do you select the right kind of technology for the job?

If I were the project lead I would first of all consult some of my peers that work as both frontend and backend developers to see what their opinions are on emerging tech. I’m very fortunate that some members of my Twitch community have been working as paid front/back end developers for a number of years. In addition to this peer research I would search on the Internet to see if there was anything that caught my eye and if their was I would email the company behind the tool to enquire about it further.

Once I’ve had a few languages and tools together I would do some very small user testing before going back to the drawing board based on how well we can get the tools to function with their needs and the overall project needs in mind.

Describe and provide analysis on how your chosen technologies have helped you achieve your outcomes. How has Trello helped you plan and how can git and GitHub help with your codes version control?

Html, CSS and javascript are deeply intertwined with website development to the point where it’s hard to have a good looking website without all of those elements working together properly. Html [4] is primarily responsible for the coding of the website where as CSS [5] is more focused on the graphical appearance of the website EG making an image change aspect ratio and more. Finally, there is javascript [6] which can be used for number of things EG detecting the use of an adblocker running which could that force the user to disable it before using the website.

Trello I’ve found has been largely redundant for me as I tend to keep things in mind but overall I see it as a useful tool when there are more than one person working on a project. I’ve mostly used it to remind me of what work I need to when finishing for the day!

Git and Github are very useful tools to push the code I’ve been working on to the cloud so that I can work on it from any machine. This also works as a good redundancy measure to protect against hardware failure and accidental loss of code providing best practice is in place!
